# Saalim Links Page — Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a polished link-in-bio page at saalim.dev using Next.js + Tailwind CSS v4, matching the dark slate + purple accent aesthetic of saalimalmarhubi.com, deployed as a static export to Cloudflare Pages.

**Architecture:** Single-page static Next.js app (App Router, `output: 'export'`). All content lives in a typed config file (`lib/links.ts`). Page composed of header (avatar + name + tagline), two link sections (connect + work), a QR code toggle, and a footer. CSS-only animations with `prefers-reduced-motion` support. No database, no CMS, no client-side routing.

**Tech Stack:** Next.js 15 (App Router), Tailwind CSS v4, TypeScript, `qrcode` npm package (lightweight, no canvas dependency issues), Google Fonts (Urbanist).

## File Structure

```
saalim-links/
├── src/
│   ├── app/
│   │   ├── layout.tsx          — Root layout: font loading, metadata, body wrapper
│   │   ├── page.tsx            — Main page: assembles all sections
│   │   └── globals.css         — Tailwind directives, custom keyframes, noise overlay, grain
│   ├── components/
│   │   ├── Avatar.tsx          — "SA" initials ring with purple gradient border
│   │   ├── Header.tsx          — Name + tagline + avatar composition
│   │   ├── LinkCard.tsx        — Individual link card with hover fill animation
│   │   ├── LinkSection.tsx     — Section label + list of LinkCards
│   │   ├── QRCodeToggle.tsx    — Client component: toggle button + QR canvas (uses 'use client')
│   │   ├── Footer.tsx          — Name, location, email
│   │   └── Icons.tsx           — All inline SVG icon components
│   └── lib/
│       └── links.ts            — Typed link data config
├── public/                       — Static assets (empty initially; SVG favicon via metadata)
├── next.config.ts              — Static export config
├── postcss.config.mjs          — PostCSS for Tailwind v4 (generated by create-next-app)
├── tsconfig.json               — TypeScript config
├── package.json                — Dependencies
└── README.md                   — Deploy instructions
```

**Design decisions:**
- Components split by responsibility (Avatar, Header, LinkCard, etc.) for clarity. Each is small (~30-80 lines).
- Only `QRCodeToggle.tsx` needs `'use client'` — everything else is server-rendered.
- Icons in a single file to avoid 6+ tiny files for simple SVGs.
- No `tailwind.config.ts` needed — Tailwind v4 uses CSS-based configuration via `@theme` in `globals.css`.

## Task 4: LinkCard Component with Hover Fill Animation

**Files:**
- Create: `src/components/LinkCard.tsx`

- [ ] **Step 1: Create LinkCard component**

Create `src/components/LinkCard.tsx`:

```tsx
import { Icon } from "./Icons";
import type { Link } from "@/lib/links";

export function LinkCard({ link, delay }: { link: Link; delay: number }) {
  const isFeatured = link.featured;

  return (
    <a
      href={link.href}
      target="_blank"
      rel="noopener noreferrer"
      className={`animate-fade-up group relative flex items-center gap-4 overflow-hidden rounded-xl border p-4 transition-all duration-200 hover:-translate-y-0.5 hover:shadow-lg hover:shadow-purple-500/5 ${
        isFeatured
          ? "border-emerald-500/15 bg-gradient-to-br from-emerald-500/8 to-slate-800"
          : "border-slate-700 bg-slate-800 hover:border-slate-600 hover:bg-slate-800/80"
      } delay-${delay}`}
    >
      {/* Purple hover fill — slides up from bottom */}
      <div
        className="absolute inset-0 translate-y-full bg-purple-300/[0.04] transition-transform duration-300 ease-out group-hover:translate-y-0"
        aria-hidden="true"
      />

      {/* Icon */}
      <div
        className={`relative z-10 flex h-10 w-10 shrink-0 items-center justify-center rounded-lg ${link.iconTheme}`}
      >
        <Icon name={link.icon} />
      </div>

      {/* Text */}
      <div className="relative z-10 min-w-0 flex-1">
        <div className="flex items-center gap-2">
          <span className="font-bold text-slate-100">{link.title}</span>
          {link.badge && (
            <span className="rounded-full bg-emerald-500/12 px-2 py-0.5 text-xs font-medium uppercase tracking-wide text-emerald-400">
              {link.badge}
            </span>
          )}
        </div>
        <p className="text-sm text-slate-400">{link.description}</p>
      </div>

      {/* Chevron */}
      <div className="relative z-10 text-slate-500 transition-all duration-200 group-hover:translate-x-1 group-hover:text-purple-400">
        <Icon name="chevron" />
      </div>
    </a>
  );
}
```

Key details:
- The purple fill div is `absolute inset-0 translate-y-full` and transitions to `translate-y-0` on `group-hover`, matching the portfolio's nav hover fill pattern.
- `overflow-hidden` on the card clips the fill element.
- `z-10` on content layers it above the fill.
- Featured card gets distinct gradient background and emerald border.
- Stagger delay applied via class `delay-N`.

- [ ] **Step 2: Create QRCodeToggle client component**

Create `src/components/QRCodeToggle.tsx`:

```tsx
"use client";

import { useState, useEffect } from "react";

export function QRCodeToggle() {
  const [show, setShow] = useState(false);
  const [qrSrc, setQrSrc] = useState<string | null>(null);

  useEffect(() => {
    if (!show) return;

    let cancelled = false;

    import("qrcode").then((mod) => {
      const QRCode = mod.default ?? mod;
      QRCode.toDataURL(window.location.href, {
        width: 200,
        margin: 2,
        color: {
          dark: "#0f172a",
          light: "#ffffff",
        },
      }).then((url: string) => {
        if (!cancelled) setQrSrc(url);
      });
    });

    return () => {
      cancelled = true;
    };
  }, [show]);

  return (
    <div className="animate-fade-up delay-7 flex w-full flex-col items-center gap-4">
      <button
        onClick={() => setShow(!show)}
        aria-expanded={show}
        className="rounded-lg border border-slate-700 bg-slate-800 px-4 py-2 text-sm text-slate-300 transition-colors duration-200 hover:border-slate-600 hover:text-slate-100"
      >
        {show ? "Hide QR Code" : "Show QR Code"}
      </button>

      {show && (
        <div className="flex flex-col items-center gap-2 rounded-xl bg-white p-4">
          {qrSrc ? (
            <img src={qrSrc} alt="QR code for this page" width={200} height={200} />
          ) : (
            <div className="flex h-[200px] w-[200px] items-center justify-center">
              <span className="text-sm text-slate-400">Loading…</span>
            </div>
          )}
          <p className="text-xs text-slate-500">Scan to open this page</p>
        </div>
      )}
    </div>
  );
}
```

Key details:
- `'use client'` — this is the only client component.
- Dynamically imports `qrcode` on toggle to avoid bundling it for users who never click.
- Uses `toDataURL` to render QR as an image (no canvas element needed in DOM).
- Cleanup function prevents state update on unmounted component.
